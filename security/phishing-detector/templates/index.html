    Le fonctionnement du modèle

    Ce détecteur utilise un Random Forest (forêt aléatoire), un algorithme d'ensemble qui combine 100 arbres de décision entraînés sur 15 features extraites de chaque URL.

    Les features les plus discriminantes sont : la présence d'une IP, l'utilisation de HTTPS, les mots suspects dans l'URL, les extensions de domaine suspectes, et la longueur de l'URL.

    Le modèle retourne une probabilité pour chaque classe (Légitime / Phishing) et affiche les features qui ont le plus contribué à la décision.
